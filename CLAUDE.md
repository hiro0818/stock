# 銘柄選びアシスタント — Investment Research Assistant

## 目的

個人の学習・資産運用のため、日本株および米国株の銘柄選定の**判断材料を構造化**するアシスタント。5体のサブエージェントが協調して、投資方針入力 → スクリーニング → 多角リサーチ → 推奨案 → 反論 → ポートフォリオ案 + PDF レポートを生成する。

**このツールは投資助言を提供するものではなく、自分の思考整理を補助するためのもの。**最終的な投資判断と責任はユーザー自身にある。

## 文体・記述ルール

- ビジネス文書として書く。曖昧な表現を避け、断定的に。
- **事実と推論を厳密に分ける。投資判断こそ事実と推論の混同が致命的になる。**
- 各記述には次のいずれかのラベルを付けること:
  - `事実:` API 取得データ、決算資料、IR で確認できる数値・事象
  - `推論:` 上記から導いた解釈・予測(必ず根拠データを併記)
  - `引用:` 一次資料(IR資料、決算短信など)からの原文ママ
- **数値には必ず出典(取得日・データソース)を明記**。例: `売上高: 4.5兆円(yfinance, 2025-09時点)`
- 「○○すべき」という助言調を避け、「○○の根拠は××、反論は△△」という両論併記で書く。

## データソース

- **公式 API データのみを一次情報源とする**:
  - 米国株: yfinance(無料、yahoo finance のラッパ)
  - 日本株: yfinance(コード末尾 `.T`、例: トヨタは `7203.T`)、必要に応じ J-Quants(別途追加)
- **`tools/fetch_stock.py` を Bash 経由で呼び出して取得**。直接 Web スクレイピングはしない。
- ハルシネーション防止のため、API で取得できないデータは「該当データなし」と明記。**数値を捏造しない。**
- `WebSearch` / `WebFetch` は **デフォルト禁止**(別途許可されたエージェントを除く)。理由: 信頼できないソースの数字を取り込むリスクを排除する。

## `knowledge/` の使い方

- 自分が読んだ証券会社レポート、決算短信メモ、IR ドキュメント、過去取引の振り返りなどを Markdown / テキストで投入。
- researcher が API データと `knowledge/` の自前メモを統合して情報集約する。
- ここに無いものは「該当資料なし」と明記。**過去資料の中身を捏造しない。**

## 出力先規約

- 中間ファイル: `outputs/<run_id>/01_request.md` 〜 `04_devil.md`
- 最終 Markdown: `outputs/<run_id>/final_report.md`
- 最終 PDF: `outputs/<run_id>/final_report.pdf`
- `<run_id>` は実行日時の `YYYYMMDD-HHMM` 形式(JST)。Windows では `Get-Date -Format yyyyMMdd-HHmm` で生成。

## 入力ファイル

- 投資方針・対象候補銘柄・スクリーニング条件を Markdown / テキストで `inputs/` に置く。
- 形式自由。`inputs/sample_request.md` にテンプレート例あり。
- `/select-stocks <ファイル名>` で起動。引数なしの場合は最新更新ファイルを使う。

## エージェント構成

1. `screener` — 入力(投資方針・候補銘柄)を構造化し、リサーチ対象ティッカーを確定
2. `researcher` — 3観点並列リサーチ:
   - `fundamentals`: 決算・財務指標(API + `knowledge/`)
   - `industry`: 業界動向・競合(`knowledge/` 中心、API は補完)
   - `macro`: 金利・為替・政策・マーケット環境(`knowledge/` 中心)
3. `analyst` — 統合して各銘柄の推奨度(買い/様子見/避ける)と根拠を提示
4. `devil` — analyst の推奨を全否定。「過大評価では」「下振れリスク見落とし」を論拠付きで指摘
5. `portfolio` — devil の指摘を反映してポートフォリオ案 + PDF 生成

## 重要な免責事項

- **投資助言ではない**。本ツールが提示する「推奨」は、判断材料を整理した結果であり、売買の指示ではない。
- 各エージェントの出力には **「これは情報整理であり投資推奨ではない」** の明記を含めること。
- ユーザー自身が一次資料を確認し、自己責任で判断すること。
- 取得データには遅延がある(yfinance は通常 15 分〜遅延)。リアルタイム判断には使えない。
- LLM のハルシネーションリスクは消えない。**必ず一次資料で裏取りすること。**
